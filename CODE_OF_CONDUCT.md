> 艹，别看到"行为准则"就觉得变得严肃了！
> 这玩意儿就是为了让咱小团队干活更舒服，别tm因为一些SB事儿影响开发心情！

---

## 📋 我们的承诺

为了让AutoGeo项目成为一个开放、包容、友好的开发社区，我们承诺：

```
✅ 尊重每个贡献者，无论技术水平高低
✅ 欢迎不同背景的开发者参与
✅ 营造积极向上的讨论氛围
✅ 对建设性意见保持开放态度
✅ 专注于解决问题，而不是人身攻击
```

---

## 🎯 适用范围

本行为准则适用于：

```
✅ GitHub Issue讨论
✅ Pull Request评论和审查
✅ 项目文档和代码注释
✅ 团队会议和讨论
✅ 任何与项目相关的沟通
```

---

## ✅ 我们鼓励的行为

### 积极参与

```markdown
✅ 使用友善和包容的语言
✅ 尊重不同的观点和经验
✅ 优雅地接受建设性批评
✅ 关注对社区最有利的事情
✅ 对其他社区成员表示同理心
```

### 技术讨论

```markdown
✅ 对事不对人（讨论代码，不讨论人）
✅ 用数据和事实说话
✅ 给出具体的改进建议
✅ 承认自己的错误和局限
✅ 感谢他人的帮助和贡献
```

### 代码审查

```markdown
✅ 建设性批评，指出问题并给出建议
✅ 解释为什么需要修改（而不是直接说"改这个"）
✅ 发现好代码时给予赞赏和认可
✅ 对新手保持耐心，帮助他们成长
✅ 尊重原作者的代码风格（只要不违反规范）
```

## 🔥 冲突解决

### 遇到不当行为怎么办

如果你遇到了不当行为，或觉得某人违反了行为准则：

1. **冷静处理**
   - 不要在公开场合争吵
   - 不要进行人身攻击反击
   - 保持专业态度

2. **举报渠道**
   - 私信项目Owner: 小a
   - 发送邮件至: [项目邮箱]
   - GitHub Issue举报（标记为私人issue）

3. **处理流程**
   ```
   收到举报
      ↓
   调查情况
      ↓
   私下沟通
      ↓
   警告或处理
      ↓
   反馈结果
   ```

### 处理措施

根据违规严重程度，可能采取以下措施：

```markdown
⚠️ 第一次违规: 口头警告
⚠️ 第二次违规: 暂时禁止参与项目（7天）
⚠️ 第三次违规: 永久禁止参与项目
⚠️ 严重违规: 直接永久禁止，无需警告
```

## 💬 沟通指南

### 技术分歧处理

当在技术方案上产生分歧时：

```markdown
1. 在Issue中列出各自的优缺点
2. 用代码测试验证（代码不会撒谎）
3. 参考社区最佳实践
4. 项目Owner最终拍板
5. 少数服从多数，保持团队团结
```

### Code Review分歧处理

```markdown
1. 先弄清楚对方的观点
2. 用事实和数据支持你的观点
3. 不决就拉第三人评审
4. 实在不行项目Owner拍板
5. 争论结束后翻篇，不记仇
```

### 友善讨论示例

```markdown
❌ 不好的说法: "这个设计太蠢了"
✅ 好的说法: "我觉得这里可以考虑另一种方案，因为..."

❌ 不好的说法: "你不懂就别瞎说"
✅ 好的说法: "我的理解是...，可能我理解有误，你可以再解释一下吗？"

❌ 不好的说法: "这种错误都能犯？"
✅ 好的说法: "这里有个潜在问题，建议加上空值检查"
```

## 📜 项目文化

### 我们的价值观

```markdown
1. 技术至上
   - 用代码说话，用事实说话
   - 不搞办公室政治，不拉帮结派

2. 简单直接
   - 有话直说，别tm弯弯绕绕
   - 对事不对人，讨论不伤感情

3. 持续改进
   - 代码可以批评，技术可以讨论
   - 目标是把项目做好，而不是证明自己正确

4. 快乐开发
   - 写代码是快乐的，别tm搞得太严肃
   - 该认真时认真，该放松时放松
```

### 团队氛围

```markdown
✅ 技术讨论激烈但不伤感情
✅ 代码批评严厉但不针对个人
✅ 帮助新手不图回报
✅ 承认错误不丢人
✅ 学习分享常态化
```

## 🤝 协作原则

### DO's（推荐做法）

```markdown
✅ 代码review时指出问题并给出建议
✅ 发现好代码时给予赞赏和认可
✅ 遇到问题及时沟通，别tm憋着
✅ 尊重别人的时间，PR描述写清楚
✅ 响应及时，别tm让人等太久
✅ 承认错误，快速修复
✅ 帮助团队成员成长
```

### DON'Ts（禁止做法）

```markdown
❌ 人身攻击，不管是公开还是私下
❌ 嘲笑新手的错误
❌ Code Review时只批评不给出建议
❌ 团伙排挤某个成员
❌ 拖着不响应PR或Issue
❌ 争论时翻旧账
❌ 背后说别人坏话
```
